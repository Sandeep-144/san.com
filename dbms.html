Database Management Systems

Database System Architecture

This section introduces the foundational architecture of database systems, covering:

  • Data Abstraction: Levels of data abstraction including physical, logical, and view levels.
  • Data Independence: The concept of logical and physical data independence.
  • Data Definition Language (DDL): Language used to define database schema.
  • Data Manipulation Language (DML): Language used for querying and updating data.

Data Models

  • Entity-Relationship Model: A graphical approach to database design using entities and relationships.
  • Network Model: A flexible way of representing objects and their relationships.
  • Relational and Object-Oriented Models: Representation of data using relations (tables) and objects.
  • Integrity Constraints: Rules to maintain the correctness of data.
  • Data Manipulation Operations: Basic operations such as insertion, deletion, and updates.

Relational Query Languages

  • Relational Algebra: A procedural query language for relational databases.
  • Tuple and Domain Relational Calculus: Non-procedural query languages for specifying queries.
  • SQL3: Extended SQL with advanced features for object-relational databases.
  • Open Source and Commercial DBMS: Examples include MySQL, Oracle, DB2, SQL Server.

Relational Database Design

  • Domain and Data Dependency: Constraints between different domains in a database.
  • Armstrong's Axioms: A set of rules used to infer all the functional dependencies on a relational database.
  • Normal Forms: Guidelines for database normalization to reduce redundancy.
  • Dependency Preservation: Ensuring that functional dependencies are maintained after decomposition.
  • Lossless Design: Decomposition of database schemas without data loss.

Query Processing and Optimization

  • Evaluation of Relational Algebra Expressions: Strategies for executing queries.
  • Query Equivalence: Equivalence of different relational algebra expressions.
  • Join Strategies: Techniques for performing joins in relational databases.
  • Query Optimization Algorithms: Algorithms to optimize the performance of queries.

Storage Strategies

  • Indices: Structures that improve the speed of data retrieval.
  • B-Trees: Balanced tree data structures for maintaining sorted data and search operations.
  • Hashing: A technique for direct data access using a hash function.

Transaction Processing

  • Concurrency Control: Methods to ensure the correctness of transactions.
  • ACID Properties: Atomicity, Consistency, Isolation, Durability properties of a transaction.
  • Serializability of Scheduling: Ensuring that transactions are executed in a serial order.
  • Locking and Timestamp-Based Schedulers: Techniques to manage transaction execution.
  • Multi-version and Optimistic Concurrency Control: Different approaches to manage concurrent transactions.
  • Database Recovery: Methods to recover databases from failures.

Database Security

  • Authentication, Authorization, and Access Control: Ensuring secure access to the database.
  • DAC, MAC, and RBAC Models: Discretionary, mandatory, and role-based access control models.
  • Intrusion Detection: Monitoring systems to detect unauthorized access.
  • SQL Injection: Preventing malicious SQL code execution.

SQL Concepts

  • Basics of SQL: Data definition and manipulation in SQL.
  • Constraints: Defining primary key, foreign key, unique, and other constraints.
  • Aggregate and Built-in Functions: Functions for numeric, date, and string operations.
  • Subqueries and Joins: Writing complex queries with subqueries and joins.
  • Views: Creating and managing views in SQL.
  • Transaction Control Commands: Commands to manage database transactions.

PL/SQL Concepts

  • Cursors: Managing database result sets using cursors.
  • Stored Procedures and Functions: Precompiled routines for reuse.
  • Database Triggers: Automated execution of code in response to certain events.
